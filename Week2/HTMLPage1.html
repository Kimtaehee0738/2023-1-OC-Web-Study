add : 어떤 가상의 공간인 Staging area에 변화를 쌓는 것 commit : 깃에서 관리하는 어떤 가상의 저장소인 Local repository에 Stagig area에 있는 것들을 결합해서 내가 최종적으로 완성하고자 하는 결과물을 저장하는 과정 push : 내 컴퓨터 안에서 commit한 내용(변화)을 remote repository(깃허브 같은 외부의 저장소)에 보냄 pull과 fetch의 차이점: pull은 자동으로 다운로드하고 병합하지만 fetch는 변경내용을 병합하기 전에 검사하고 병합의 여부를 결정할 수 있다.
